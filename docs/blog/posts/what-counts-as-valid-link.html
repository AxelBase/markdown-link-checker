What Counts as a Valid Link?

Published: September 26, 2025 | Last Updated: September 26, 2025

Not all links are created equal. Our Markdown Link Checker uses clear, rule-based logic to determine which links are valid, which are broken, and which should be skipped or flagged. This post explains how the tool interprets different link types and what you can expect from the results.

Link Types We Validate

  • HTTP/HTTPS URLs: These are fully validated with status codes and redirect tracking.
  • Relative paths: Checked against a virtual file tree if defined, useful for internal documentation.
  • Anchor links: Syntax is validated, but target existence requires a defined structure.
  • Reference-style links: Parsed and resolved before validation.

Skipped or Ignored Links

Some links are intentionally skipped to avoid false positives or unnecessary checks:

  • mailto: and tel: links
  • localhost or internal dev URLs
  • Custom schemes (e.g., ftp://, chrome://)

You can override this behavior using the ignore pattern field or by toggling external-only validation.

Validation Logic

Each link is checked using a lightweight request. Redirects are followed, and the final status code is recorded. Broken links are flagged with error codes (e.g., 404, timeout), while valid links return 200 or similar success codes. Redirects are not considered broken but are highlighted for review.

FAQ

Are redirects considered valid?

Yes, but they’re flagged separately so you can decide whether to update them.

Can I ignore certain domains?

Yes. Use the ignore pattern field to exclude domains or paths from validation.

What about anchor links?

We check anchor syntax but can’t verify target existence unless a virtual tree is defined.
